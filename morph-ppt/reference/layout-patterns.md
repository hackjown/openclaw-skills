# Layout Patterns

Choose layouts per slide based on content, not one fixed template.

## Common layouts

### 1. hero-cover
Use for opening slide.
- large title
- concise subtitle
- one strong image or graphic anchor
- 1 short learning objective block is optional

### 2. left-text-right-image
Use for concept explanation.
- title + subtitle on top-left
- 3–5 bullets on left
- illustration on right

### 3. top-text-bottom-image
Use when the image or diagram needs width.
- title + subtitle on top
- short bullets above or beside image
- wide image below

### 4. three-cards
Use for three pillars / three capabilities / three steps.
- short intro at top
- 3 parallel cards in middle
- optional small icons or mini illustrations

### 5. process-flow
Use for workflows or trigger logic.
- title at top
- 3–5 steps arranged horizontally or vertically
- arrows / connectors / grouped blocks
- optional support image

### 6. comparison
Use for before vs after, option A vs B, old vs new.
- two balanced columns
- same hierarchy on both sides
- one takeaway sentence

### 7. timeline-roadmap
Use for phases or plan.
- horizontal or vertical timeline
- 3–6 milestones only
- concise labels

### 8. faq-troubleshooting
Use for problem solving slides.
- issue / cause / action pattern
- use cards or stacked rows
- prioritize clarity over decoration

### 9. summary-takeaway
Use for final page.
- one conclusion sentence
- 3 bullets max
- optional next-step box

## Selection rules

- If the slide explains a concept -> `left-text-right-image`
- If the slide is dominated by a visual diagram -> `top-text-bottom-image`
- If the slide has 3 parallel ideas -> `three-cards`
- If the slide explains a workflow -> `process-flow`
- If the slide compares options -> `comparison`
- If the slide is about phases or rollout -> `timeline-roadmap`
- If the slide addresses common issues -> `faq-troubleshooting`
- If the slide closes the deck -> `summary-takeaway`

## Rule

In `brief.md`, every slide must specify one layout explicitly.
Avoid using the same layout on every page unless the user asks for strict consistency.
